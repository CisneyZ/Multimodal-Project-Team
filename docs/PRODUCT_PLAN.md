# 产品计划

## 产品愿景

让公司员工能够在 Web 或飞书中安全地调用多个专业 Agent，并获得可追踪、可复核、可复用的结构化结果。

## 产品形态

```text
Web 管理后台：完整配置、任务、审核、知识库、模型与工具管理
飞书：免登入口、机器人、任务卡片、审核和通知
```

## 一级导航

第一版 Web 管理后台包含：

- 工作台
- Agent 中心
- 任务中心
- 审核中心
- 知识库
- 模型与工具
- 系统设置

## 第一版核心用户流程

```text
用户在 Web/飞书提交文本或文件
↓
系统创建 Task
↓
Agent 执行并记录当前步骤
↓
Model Gateway 或 Tool 执行
↓
结构化校验
↓
必要时进入人工审核
↓
结果返回 Web/飞书
```

## 第一个 Agent：影视人才评测

当前闭环：

```text
项目资料导入
↓
简历结构化
↓
项目匹配
↓
动态测评
↓
AI 初评 + 人工复核
↓
评估报告
↓
人才库
```

当前阶段只迁移“简历结构化分析”，其他流程暂不重写。

## 第二个 Agent：Prompt 质检

选择原因：

- 输入主要是文本，开发成本低。
- 规则明确，结果容易验证。
- 能检验 Agent、Workflow、Knowledge、Model Profile 和审核机制是否真的可复用。

## 产品成功指标

第一版重点不是用户量，而是稳定性：

- 任务成功率
- 结构化输出通过率
- API 平均延迟
- Provider 失败率
- 备用模型启用率
- 人工复核率
- 飞书任务回传成功率
- 单任务成本

## 第一版范围外

- 公开注册和商业计费
- 多租户 SaaS
- 大型素材资产库
- 自主无限循环 Agent
- 全自动替代人工审核
