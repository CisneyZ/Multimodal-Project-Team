# 飞书联动设计

## 定位

飞书是公司员工的协作入口，平台是 Agent 的执行和数据核心。

## 第一批能力

### 1. 免登

员工从飞书工作台打开 Web 应用，后端根据飞书身份映射平台用户和角色。

角色：

- user
- project_owner
- reviewer
- admin

### 2. 机器人创建任务

支持文本、文件和明确指令，例如：

```text
帮我分析这份简历
质检这个 Prompt
```

机器人只负责接收、创建 Task 和返回卡片；实际 Agent 任务异步运行。

### 3. 任务卡片

卡片显示：

- Agent
- 任务名称
- 发起人
- 当前状态
- 当前步骤
- 创建时间
- 查看详情
- 取消任务（权限允许时）

### 4. 审核卡片

进入 `waiting_review` 时通知审核人员，提供：

- 关键结论
- 风险提示
- 证据摘要
- 通过
- 驳回
- 打开完整报告

### 5. 通知

- 任务完成
- 任务失败
- 需要审核
- 模型连续失败
- 知识同步失败
- 成本超过阈值

## 多维表格

用途：

- 管理看板
- 人才名单
- 项目状态
- 审核列表
- 汇总统计

不用于保存全部 Task 节点日志、模型调用、文件关系和复杂权限。

## 知识同步

飞书文档和多维表格可以作为 Knowledge Source。同步流程：

```text
飞书变更事件
↓
同步任务
↓
拉取内容
↓
解析与切分
↓
版本化
↓
更新平台知识库
```

## 事件处理

### 本地开发

优先长连接，降低公网回调配置成本。

### 生产环境

HTTPS Webhook：

```text
事件校验与去重
↓
快速写入队列
↓
立即响应飞书
↓
Worker 异步处理
```

回调函数内禁止等待大模型分析完成。

## 安全

- App Secret 只放 `.env` 或密钥管理服务。
- 验证事件签名与来源。
- 对事件做幂等去重。
- 权限采用最小授权。
- 不在卡片中展示敏感简历全文。
