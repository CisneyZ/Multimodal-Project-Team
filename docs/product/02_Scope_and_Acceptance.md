# MPT Platform v0.1 范围与验收标准

## P0 必须完成

- 项目可本地启动，前后端无空白页和无限加载；
- 复用或实现可用登录；
- 两个内置项目可查看；
- 候选人新增、查看、编辑、软删除；
- 粘贴简历文本；
- 至少一种文件格式可稳定解析；
- 人才画像可编辑并保存；
- 同时匹配两个项目；
- 分数由规则引擎计算；
- 硬性条件独立判断；
- 主要判断显示证据或缺失原因；
- 报告由服务端生成并保存；
- 报告可复制和浏览器打印；
- AI 不可用时自动 fallback；
- 三个内置虚构案例均可跑通。

## P1 应该完成

- 文件上传状态；
- 候选人列表分页；
- 报告列表；
- 操作日志；
- 首页真实统计；
- 规则版本字段；
- 招聘状态流转。

## P2 可延后

- 完整规则编辑器；
- 大批量上传；
- 完整多角色权限；
- 对象存储；
- 飞书免登；
- PDF 服务端导出；
- 高级数据分析。

## 明确不做

- 动态试题；
- 自动面试；
- 在线试标；
- Agent 市场；
- 拖拽工作流；
- 视频理解；
- 微服务；
- Redis；
- BullMQ；
- 模型训练。

## 推荐等级

| 分数 | 等级 |
|---|---|
| 85–100 | 强烈推荐 |
| 70–84 | 推荐进入面试或试标 |
| 55–69 | 待补充信息 |
| 0–54 | 不推荐 |

覆盖规则：

- 关键硬性条件不通过：最终为条件不满足或不推荐；
- 无不通过但存在关键缺失：最终最多为待补充信息；
- 全部通过：按分数确定等级。

## 发布验收

- [ ] `npm run type:check` 通过
- [ ] `git diff --check` 通过
- [ ] 核心规则测试通过
- [ ] 三案例结果符合预期
- [ ] AI 关闭后仍能运行
- [ ] 候选人和报告刷新后存在
- [ ] 无 API Key、真实简历或日志被提交
- [ ] 页面无空白和无限加载
