# MPT Platform v0.1 测试计划

## 核心规则

- RL 高匹配：RL 高于 Caption，硬性条件全部通过；
- Caption 高匹配：Caption 高于 RL，硬性条件全部通过；
- 不合格：两项目关键硬性条件失败，最终不推荐；
- 信息缺失：最终最多待补充信息；
- 人工修正：重新生成 ScreeningRun，模式为 manual_corrected。

## 文件

- TXT；
- PDF；
- DOCX；
- 空文件；
- 超大文件；
- 不支持格式；
- 解析失败；
- 同名文件。

- 新增候选人；

- 画像编辑；
- 双项目结果；
- 硬性条件；

- 修改后重算；
- 刷新后数据存在。

## 发布前人工验收

```text
案例A → RL 推荐
```
